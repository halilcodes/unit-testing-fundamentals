***************************************
*     Unit Testing Best Practices     *
***************************************

1. Write readable unit tests.

2. Refrain from using multiple assertions in a single test.

3. Refrain from adding logic to unit tests.
    - avoid if statements 

4. Avoid test interdependencies.

5. Avoid using numeric constant values.
    - use math library's pi instead of 3.14 for example

6. Write deterministic tests.

7. Write unit tests while developing the production code.
    - TDD: test-case first, code later

8. Automate the tests using CI/CD tools.
    - TODO: add CI/CD course to finish soon.

9. Name the tests names descriptive enough to understand.
    - Clean Code
    
10. Refactor the unit tests periodically.
    - Clean Code
